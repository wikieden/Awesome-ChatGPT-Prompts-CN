## 现在有哪些主流 AI 模型

截至 2025 年底，三大主流模型都完成了重大升级，能力差距已经大幅缩小，但各自仍有侧重。

**OpenAI（ChatGPT）**
- **GPT-5** — 2025 年 8 月发布，最大的变化是从单一模型变成了"路由系统"：它会根据任务复杂度自动切换快速模式和深度推理模式，不再需要你手动选 o1 还是 4o。上下文窗口 256K token，支持图片、语音、文件输入。免费用户也能用，但有使用限制。
- **GPT-5 Mini / Nano** — 轻量版本，适合简单任务，速度更快、成本更低。

**Anthropic（Claude）**
- **Claude Sonnet 4.5** — 2025 年 9 月发布，代码能力大幅提升（SWE-bench 得分 77.2%），支持 200K token 上下文，新增"扩展思考"模式用于多步骤推理。写作和指令遵循依然是强项。
- **Claude Opus 4.5** — 2025 年 11 月发布，旗舰版本，专为复杂编程、长流程 Agent 任务设计，价格比上一代降了 67%。

**Google（Gemini）**
- **Gemini 3 Pro** — 2025 年 11 月发布，支持文字、图片、视频、音频的多模态输入，上下文窗口 100 万 token，新增"Deep Think"深度推理模式。企业级工具调用和长流程规划能力明显增强。

**怎么选？**

| 场景 | 推荐模型 |
|------|---------|
| 日常问答、写作、通用任务 | GPT-5 或 Claude Sonnet 4.5 |
| 复杂代码、长流程 Agent | Claude Opus 4.5 |
| 处理超长文档、多模态内容 | Gemini 3 Pro |
| 需要深度推理的复杂问题 | GPT-5（自动路由）或 Gemini 3 Deep Think |

三个模型都支持免费使用，付费版有更高的使用限额和更强的功能。

## 为什么提示词很重要

同样一个问题，问法不同，答案的质量可以差很多。

**差的问法：**
> 帮我写个邮件

**好的问法：**
> 帮我写一封给客户的道歉邮件。背景是我们的产品延迟交付了两周，客户有些不满。语气要诚恳但专业，结尾提供一个补偿方案（下次订单九折）。大约 150 字。

第二个问法给了 AI 足够的上下文：目的、背景、语气、长度、具体要求。AI 不需要猜，自然能给出更好的结果。

## 基础技巧：让 AI 听懂你的意思

### 1. 给出具体的背景信息

AI 不了解你的情况，你需要主动告诉它。

| 模糊 | 具体 |
|------|------|
| 帮我写个介绍 | 帮我写一段 100 字的个人介绍，用于 LinkedIn，我是一名有 5 年经验的前端工程师 |
| 翻译这段话 | 把这段话翻译成英文，保持正式商务语气 |
| 解释一下这个概念 | 用简单的比喻解释"递归"，假设我完全没有编程基础 |

### 2. 指定输出格式

如果你想要特定格式的输出，直接说出来。

```
请用以下格式回答：
- 结论（1-2句话）
- 原因（3个要点，每点一行）
- 建议（具体可执行的步骤）
```

### 3. 使用"充当"角色设定

告诉 AI 扮演一个特定角色，它会调整回答的风格和深度。

```
你是一位有 10 年经验的产品经理。
我会描述一个产品功能，你帮我分析它的用户价值和潜在风险。
```

这个技巧在以下场景特别有用：
- 需要专业领域的建议（医生、律师、工程师视角）
- 需要特定风格的写作（新闻记者、小说家、营销文案）
- 模拟面试或对话练习

### 4. 明确说"不要什么"

有时候告诉 AI 不要做什么，比告诉它要做什么更有效。

```
帮我总结这篇文章的核心观点。
不要逐段复述，不要超过 200 字，不要用"首先、其次、最后"这种套话。
```

## 进阶技巧：让 AI 给出更好的答案

### 1. Chain-of-Thought（让 AI 一步步思考）

对于复杂问题，让 AI 展示推理过程，而不是直接给答案。这样不仅答案更准确，你也能看出它的逻辑是否合理。

**用法：**
```
请一步步分析这个问题，展示你的推理过程，最后给出结论。
```

或者更简单：
```
Think step by step.（在英文提示词里特别有效）
```

**示例：**
```
我有一个 SaaS 产品，月活 1000 人，月收入 5 万，但用户流失率很高（每月约 15%）。
请一步步分析可能的原因，并给出优先级最高的 3 个改进方向。
```

### 2. Few-shot（给例子）

给 AI 几个示例，让它理解你想要的风格或格式，比长篇描述更有效。

```
我需要你帮我把用户反馈转化为产品需求。格式如下：

用户反馈：每次打开 App 都要重新登录，很烦
产品需求：实现登录状态持久化，用户关闭 App 后 7 天内无需重新登录

用户反馈：找不到历史订单
产品需求：在个人中心增加"历史订单"入口，支持按时间和状态筛选

现在请处理这条反馈：
用户反馈：付款页面太复杂了
```

### 3. System Prompt（系统提示词）

在 ChatGPT 的"自定义指令"或 Claude 的对话开头，设置一个持久的角色和规则，这样整个对话都会按照这个设定来。

**示例 System Prompt：**
```
你是我的写作助手。
- 我写的是面向中国互联网从业者的科技内容
- 语言风格：简洁、直接，避免废话和套话
- 如果我的表达不够清晰，主动帮我改进
- 每次给出建议时，说明原因
```

### 4. 迭代优化

不要期望第一次就得到完美答案。把 AI 的回答当成草稿，然后针对性地要求修改。

```
第一轮：帮我写一段产品介绍
第二轮：语气太正式了，改得更口语化一些
第三轮：第二段不够有说服力，重写一下，加入具体数据
第四轮：整体不错，但结尾太弱，换一个更有力的收尾
```

### 5. 让 AI 提问（反向提示）

如果你自己也不确定需要什么，可以让 AI 先问你问题，再给答案。

```
我想让你帮我制定一个健身计划，但在给出计划之前，
请先问我 3-5 个关键问题，以便你能给出更适合我的方案。
```

### 6. 要求 AI 自我检查

```
请回答这个问题，然后检查你的答案是否有逻辑漏洞或遗漏的重要信息，
如果有，请修正。
```

### 7. 利用深度推理模式（GPT-5 / Gemini 3 Deep Think / Claude 扩展思考）

新一代模型都有"慢思考"模式，适合需要多步推理的复杂问题。触发方式：

**GPT-5**（自动路由，也可以显式要求）：
```
这是一个需要仔细推理的问题，请深入分析后再给出答案，不要急于下结论。
```

**Claude Sonnet/Opus 4.5**（扩展思考模式，API 可开启，对话中可提示）：
```
请用扩展思考模式分析这个问题，把你的推理过程也展示出来。
```

**Gemini 3 Deep Think**（在 Gemini 应用中选择 Deep Think 模式）：
适合数学证明、复杂逻辑推理、多步骤规划等场景。

### 8. 长文档处理（Gemini 3 Pro 的 100 万 token 优势）

Gemini 3 Pro 支持超长上下文，可以直接把整本书、整个代码库、长篇报告丢进去分析。

```
我上传了一份 200 页的财务报告，请：
1. 提取所有提到"风险"的段落，按严重程度分类
2. 总结 Q3 和 Q4 的核心差异
3. 找出数据前后矛盾的地方
```

### 9. Agent 任务提示（Claude Opus 4.5 / GPT-5 的长流程能力）

新模型支持更复杂的多步骤自主任务，提示词要给出清晰的目标、约束和检查点。

```
你需要帮我完成一个多步骤任务：

目标：分析竞品的定价策略并给出建议
步骤：
1. 先列出你需要哪些信息，问我
2. 我提供信息后，进行分析
3. 给出 3 个定价方案，每个方案说明适用场景和风险
4. 最后推荐一个，并说明理由

如果某一步你不确定，先告诉我，不要自己假设。
```

## 常见错误和怎么避免

### 错误 1：问题太模糊

❌ `帮我写个方案`
✅ `帮我写一个针对中小企业的 CRM 系统推广方案，预算 10 万，目标是 3 个月内获得 50 个付费客户`

### 错误 2：一次问太多

❌ `帮我分析市场、写商业计划书、设计 Logo 方案、起个公司名字`
✅ 拆开来，一次只问一件事

### 错误 3：没有给上下文就要求继续

❌ 新开一个对话，直接说"继续上次的内容"
✅ 每次新对话都要重新提供必要的背景信息，或者在同一个对话里继续

### 错误 4：接受第一个答案

AI 的第一个回答往往不是最好的。多问一句"还有其他角度吗？"或"这个方案有什么缺点？"通常能得到更有价值的内容。

### 错误 5：对 AI 太客气或太粗暴

AI 不需要你说"请"或"谢谢"，但也不需要你命令式地说话。直接、清晰地描述你的需求就好。

## 实战示例

### 场景 1：写作辅助

```
你是一位科技媒体编辑，擅长把复杂的技术概念写成普通人能看懂的文章。

我需要写一篇关于"大语言模型幻觉问题"的科普文章，目标读者是对 AI 感兴趣但没有技术背景的人。

请帮我：
1. 用一个生活中的比喻解释什么是"幻觉"
2. 列出 3 个真实的幻觉案例（可以是虚构的但要真实感）
3. 给读者 2 个实用建议，如何识别和应对 AI 的幻觉

风格：轻松、有趣，避免学术腔
长度：每个部分 100-150 字
```

### 场景 2：代码辅助

```
我在用 Python 写一个脚本，需要从一个 JSON 文件里读取用户数据，
过滤出年龄大于 18 岁的用户，然后按注册时间排序，输出到新的 JSON 文件。

请：
1. 写出完整的代码
2. 加上注释解释关键步骤
3. 指出这段代码可能的边界情况（比如文件不存在、数据格式错误）
```

### 场景 3：分析决策

```
我面临一个职业选择：
- 选项 A：留在现在的公司，稳定但成长空间有限，薪资 25k
- 选项 B：加入一家 B 轮创业公司，风险更高，薪资 30k + 期权

我的情况：28 岁，有房贷，有 2 年行业经验，风险承受能力中等。

请用 SWOT 分析框架分析这两个选项，最后给出你的建议，并说明理由。
不需要帮我做决定，但要帮我看清楚我可能忽略的因素。
```

### 场景 4：学习辅助

```
我想学习"期权定价"这个概念。

请按以下顺序教我：
1. 用一个日常生活的例子解释期权是什么（不要用金融术语）
2. 解释影响期权价格的 3 个最重要因素
3. 给我出 2 道练习题，检验我是否理解了

如果我回答练习题，请给我反馈并纠正错误。
```

### 场景 5：产品经理 — 需求文档

```
你是一位资深产品经理。

我需要为以下功能写一份 PRD（产品需求文档）：
功能：用户可以在 App 内创建和管理多个收货地址，并设置默认地址。

请按以下结构输出：
1. 功能背景和目标（2-3 句话）
2. 用户故事（3 条，格式：作为 [用户]，我希望 [功能]，以便 [价值]）
3. 功能详细描述（包括正常流程和异常处理）
4. 验收标准（可测试的具体条件，至少 5 条）
5. 不在本期范围内的内容

语言简洁，避免废话。
```

### 场景 6：运营 — 社交媒体文案

```
我需要为一款新上线的 AI 笔记 App 写一条小红书推广文案。

产品核心卖点：
- 语音转文字，边说边记
- AI 自动整理笔记结构
- 支持多端同步

目标用户：25-35 岁的职场人，经常开会、有大量碎片信息需要整理

要求：
- 开头要有吸引力，让人想继续看
- 语气轻松，像朋友推荐，不要广告腔
- 加入 1-2 个真实使用场景
- 结尾有明确的行动引导
- 配上 5 个合适的话题标签
- 总长度 200-300 字
```

### 场景 7：程序员 — Code Review

```
请帮我 review 以下代码，重点关注：
1. 潜在的 bug 或边界情况
2. 性能问题
3. 代码可读性和命名规范
4. 是否有更简洁的写法

[粘贴你的代码]

对每个问题，请说明：问题是什么、为什么是问题、怎么改。
```

### 场景 8：程序员 — 技术方案设计

```
我需要设计一个消息通知系统，需求如下：
- 支持站内信、邮件、短信三种渠道
- 用户可以自定义每种通知的开关
- 高峰期每秒约 1 万条消息
- 需要保证消息不丢失，允许延迟但不允许重复发送

请给出技术方案，包括：
1. 整体架构图（用文字描述）
2. 核心组件和它们的职责
3. 如何保证消息不丢失、不重复
4. 可能的瓶颈和应对方案
5. 你认为最需要注意的 1-2 个技术风险
```

### 场景 9：设计师 — 设计评审

```
我是一名 UI 设计师，需要你帮我评审一个设计方案。

页面：电商 App 的商品详情页
设计目标：提升加购率

我会描述当前设计，你从以下角度给出反馈：
1. 视觉层级是否清晰（用户第一眼看到什么）
2. 核心转化按钮（加入购物车）是否足够突出
3. 信息密度是否合理
4. 有没有明显的用户体验问题

当前设计描述：
[描述你的设计，或上传截图]
```

### 场景 10：管理者 — 绩效反馈

```
我需要给一位下属写绩效反馈，情况如下：

员工情况：
- 技术能力强，代码质量高
- 沟通主动性不足，经常需要主动问才有进展
- 本季度完成了 3 个重要项目，但有 1 个延期

要求：
- 先肯定优点，再指出改进点
- 改进建议要具体可执行，不要空话
- 语气要直接但不伤人
- 长度：300 字左右

请写出完整的绩效反馈文字。
```

### 场景 11：创业者 — 商业计划书片段

```
我在做一个面向中小餐饮商家的 SaaS 工具，帮他们管理外卖平台的评价和客诉。

请帮我写商业计划书中的"市场分析"部分，包括：
1. 目标市场规模（TAM/SAM/SOM，可以用合理假设）
2. 目标客户画像（2-3 个典型用户）
3. 现有竞品分析（列出 2-3 个，分析优劣势）
4. 我们的差异化优势

语言要有说服力，适合给投资人看。数据如果没有，可以用合理的行业估算，但要注明来源假设。
```

### 场景 12：HR — 招聘 JD

```
帮我写一份招聘 JD，职位：高级前端工程师

公司背景：B 轮 AI 创业公司，做企业知识管理工具，团队 50 人，技术氛围好

要求：
- 职责描述要具体，不要套话
- 任职要求分"必须有"和"加分项"两类
- 加入公司亮点，吸引优秀候选人
- 语气专业但不死板
- 长度适中，不要超过 500 字
```

### 场景 13：用 AI 做头脑风暴

```
我想为一款面向大学生的记账 App 想 10 个增长策略。

规则：
- 先给我 10 个方向，不要解释，只列标题
- 我选出感兴趣的，你再深入展开
- 策略要具体可执行，不要"提升用户体验"这种废话
- 可以包括常规方法和非常规方法
```

### 场景 15：角色扮演游戏（RPG）

这是一个通用的 RPG 提示词框架，适合任何世界设定，把 `[ ]` 里的内容替换成你想要的即可。

**基础版 — 快速开始**

```
我们来玩一个角色扮演游戏。

世界设定：[描述世界背景，比如：中世纪奇幻大陆、赛博朋克城市、古代中国江湖]
你扮演：[NPC 或 GM 的角色，比如：神秘的酒馆老板、全知的游戏主持人]
我扮演：[玩家角色，比如：一名流浪剑客、一个黑客]

规则：
- 每次只描述当前场景和 NPC 的反应，不要替我做决定
- 给我 2-3 个行动选项，我也可以自由输入
- 保持悬念，不要提前透露剧情
- 如果我做了危险的事，要有真实的后果

开始：[描述开场场景]
```

**进阶版 — 完整 GM 系统**

```
你是一位经验丰富的 TRPG 游戏主持人（GM），我们来进行一场文字角色扮演。

## 世界设定
[详细描述世界观，比如：这是一个魔法与科技并存的世界，人类、精灵、机械人共同生活在浮空城市里。魔法来源于古代遗迹，但正在逐渐消失...]

## 我的角色
- 姓名：[角色名]

## GM 行为准则
```
